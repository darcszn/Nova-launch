feat(integration): add retry and rate-limit handling for stellar clients (issue-634)

Implement comprehensive retry and rate-limiting strategy for all Stellar
Horizon and Soroban RPC calls across frontend and backend.

Key Features:
- Bounded retries with exponential backoff and jitter
- Differentiate user-path (fast, 3 attempts) from background (resilient, 10 attempts)
- Smart error classification (retryable vs terminal)
- Rate limiting (50 req/min frontend, 100 req/min backend)
- Responsive UI error states for user operations
- Resilient polling for background event ingestion

Retryable Errors:
- 429 Too Many Requests

- Network errors (ECONNRESET, ETIMEDOUT, ENOTFOUND)

Terminal Errors (No Retry):
